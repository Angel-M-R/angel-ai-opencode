# Comparable open-source agent systems

## Conclusion

Nine projects are reasonable direct comparison candidates for the README's current Gentle AI / Angel AI / Oh My Pi tables. They install a multi-stage development workflow, an orchestration layer, or both. Seven additional projects are relevant but should be presented separately because they are primarily methodologies, spec/task systems, or execution loops rather than full harness distributions.

The strongest direct candidates by breadth of comparable evidence are ECC, gstack, Ruflo, BMAD Method, oh-my-claudecode, Compound Engineering, CCPM, and GSD Core. `wshobson/agents` also qualifies because it ships explicit orchestrators and end-to-end commands, but its marketplace/catalog shape makes it the weakest direct comparison.

## Research method

- **GitHub star snapshot:** `2026-07-23T21:08:40Z` (UTC).
- **Star source:** one GitHub GraphQL API query over the 16 included canonical repositories. Each count below links to the corresponding first-party REST API endpoint, where `stargazers_count` is the equivalent field.
- **Capability sources:** only each project's current official repository README, repository docs, or source files.
- **Meaning of "not evidenced":** the reviewed first-party sources did not establish the capability. It is not a claim that the capability cannot exist.
- **Meaning of "not applicable":** the category is outside the focused tool's intended scope.
- **Direct comparison rule:** an installable distribution must cover multiple development stages and provide a coherent workflow or orchestration layer. A large prompt/agent catalog alone is insufficient.
- **Adjacent rule:** the project contributes a serious method or focused subsystem that is useful context, but putting it in the existing harness tables would imply broader ownership than its official sources support.

- **Renamed repositories:** `ruvnet/claude-flow` now resolves to `ruvnet/ruflo`, and `affaan-m/everything-claude-code` now resolves to `affaan-m/ECC`. The canonical current repositories are ranked rather than stale names.
- **Configuration/template marketplaces without one coherent workflow:** [`davila7/claude-code-templates`](https://github.com/davila7/claude-code-templates) is useful for configuring and monitoring Claude Code, but its catalog shape does not establish one integrated development lifecycle comparable to the README tables. Generic prompt/persona collections are excluded for the same reason.
- **Ordinary IDE/coding agents:** Cline, Roo Code, Kilo Code, OpenCode, Aider, Continue, and similar host agents are excluded by scope. They are runtimes or coding agents, not overlay workflow distributions being compared on these categories.
- **Awesome lists and prompt collections:** generic "awesome" repositories and repositories that only aggregate agent prompts were excluded because catalog size alone does not evidence an integrated workflow.
- **SDKs/frameworks and standalone multi-agent applications:** orchestration SDKs and systems such as MetaGPT/ChatDev use a different unit of comparison. Their runtime/framework architecture would distort tables intended for installable coding-harness overlays.

## Interpretation notes for a future README update

- Keep star counts out of capability scoring. Popularity changes quickly and does not establish workflow quality.
- Do not compare raw "agent count" without stating whether entries are fixed prompt files, dynamically spawned roles, skill-local personas, or external CLI workers.
- Do not compare raw "MCP count" without stating whether the project bundles several servers, one server exposing many tools, optional config examples, or no server because it inherits the host's tools.
- Treat file-backed state, session summaries, learned patterns, and semantic/vector memory as separate memory designs.
- Treat context reduction, model routing, budget tracking, and measured token savings as separate optimization claims.
- For direct additions to the existing tables, ECC, gstack, Ruflo, BMAD Method, oh-my-claudecode, Compound Engineering, CCPM, and GSD Core have the clearest coherent workflow identity. Add `wshobson/agents` only if a marketplace-shaped distribution is acceptable as a column.
