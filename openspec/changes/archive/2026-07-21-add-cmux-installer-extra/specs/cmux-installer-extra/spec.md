## ADDED Requirements

### Requirement: Optional cmux extra selection
The installer SHALL expose one cmux extra covering both notifications and Feed, and the interactive installer SHALL leave it unselected by default while preserving the existing defaults of other extras.

#### Scenario: Interactive installer starts with cmux unselected
- **WHEN** the installer opens the extras selection step
- **THEN** the cmux extra is visible and unselected
- **AND** all pre-existing extras retain their prior default selection state

#### Scenario: User selects the cmux extra
- **WHEN** the user explicitly selects the cmux extra
- **THEN** the installation request includes the complete cmux notifications and Feed integration

### Requirement: Pinned vendored cmux assets
Angel AI SHALL own vendored `cmux-session.js` and `cmux-feed.js` files that are exact snapshots generated by cmux 0.64.20, and the repository SHALL document that provenance.

#### Scenario: Installer prepares the pinned snapshot
- **WHEN** the selected cmux extra is prepared for installation
- **THEN** both desired file contents come from Angel AI's vendored cmux 0.64.20 snapshot
- **AND** the installer does not invoke `cmux hooks` or fetch replacement content

### Requirement: Selected cmux prerequisite preflight
The installer SHALL require the `cmux` executable to be available in `PATH` only when the cmux extra is selected, and SHALL complete this check before performing any installation mutation.

#### Scenario: Selected extra cannot find cmux
- **WHEN** the cmux extra is selected and `cmux` cannot be resolved from `PATH`
- **THEN** planning and application fail with a diagnostic that identifies the missing cmux prerequisite
- **AND** application does not install packages or write configuration files

#### Scenario: Unselected extra does not require cmux
- **WHEN** the cmux extra is not selected
- **THEN** planning and application do not require or look up `cmux`

### Requirement: Auto-discovered plugin installation
When selected, the installer SHALL plan and install both vendored files as `plugins/cmux-session.js` and `plugins/cmux-feed.js` beneath the target OpenCode configuration directory, relying exclusively on OpenCode plugin-directory auto-discovery.

#### Scenario: Plan includes both plugin files
- **WHEN** the cmux extra is selected and preflight succeeds
- **THEN** the installation plan includes the target path for each cmux plugin file

#### Scenario: Apply installs both plugin files
- **WHEN** the cmux extra is selected and application succeeds
- **THEN** both vendored files exist at their respective `plugins/` target paths
- **AND** `opencode.json` is not created or modified on behalf of the cmux extra

### Requirement: Drift-safe and idempotent reconciliation
The installer SHALL reconcile each cmux plugin through the shared file reconciliation behavior, backing up divergent existing content before replacement and leaving byte-identical desired content unchanged.

#### Scenario: Divergent plugin is backed up and replaced
- **WHEN** a target cmux plugin exists with content different from the vendored snapshot
- **THEN** the installer creates a backup containing the previous content before replacing the target
- **AND** the target contains the vendored snapshot after successful application

#### Scenario: Successful reapplication is idempotent
- **WHEN** both target cmux plugins already match the vendored snapshots
- **THEN** reapplication performs no file replacement
- **AND** reapplication creates no additional backups

#### Scenario: Retry converges after partial application
- **WHEN** an earlier application wrote one cmux plugin but did not complete the other
- **THEN** a retry leaves the matching plugin unchanged and reconciles the incomplete plugin
- **AND** both targets match the vendored snapshots after the retry succeeds

### Requirement: Deselection preserves installed cmux plugins
The installer SHALL NOT remove or modify previously installed cmux plugin files solely because the cmux extra is unselected.

#### Scenario: Previously installed extra is later unselected
- **WHEN** an installation request leaves the cmux extra unselected after its files were installed previously
- **THEN** both installed cmux plugin files remain untouched
