## Why

Angel AI users who run OpenCode inside cmux currently lack installer-managed access to cmux notifications and Feed integration. Providing a pinned, vendored integration makes this functionality reproducible without delegating ownership or installation behavior to `cmux hooks`.

## What Changes

- Add an optional cmux installer extra, unselected by default, that installs both notification and Feed plugins.
- Vendor exact `cmux-session.js` and `cmux-feed.js` snapshots generated by cmux 0.64.20 and document their provenance.
- Block installer preflight with a clear diagnostic when the extra is selected and `cmux` is not available in `PATH`.
- Copy both files into OpenCode's `plugins/` directory and rely exclusively on plugin auto-discovery without modifying `opencode.json`.
- Reconcile divergent installed files through the existing backup behavior and support idempotent retries.
- Preserve installed cmux files when the extra is later unselected; no uninstall behavior is added.
- Add installer-focused tests for selection, planning/application, preflight, backups, and idempotency.

## Capabilities

### New Capabilities
- `cmux-installer-extra`: Defines selection, preflight, installation, reconciliation, and persistence behavior for the vendored cmux notification and Feed plugins.

### Modified Capabilities

None.

## Impact

- Affects the installer's optional-extra model, preflight and plan/apply flow, bundled plugin assets, and installer tests.
- Adds `cmux` as a runtime prerequisite only when this extra is selected.
- Does not change OpenCode configuration, invoke `cmux hooks`, add uninstall behavior, or expand testing beyond installer behavior.
